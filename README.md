# GradelVPN (Flutter)

Клиент VLESS/V2Ray с поддержкой подписок. Пользователь вставляет ссылку
от твоего Telegram-бота (@gradelvpnbot), приложение загружает список
серверов, даёт выбрать и подключиться в один тап.

Under the hood — плагин [`flutter_v2ray`](https://pub.dev/packages/flutter_v2ray),
он оборачивает Xray-core. Сам протокол VLESS с нуля не пишется — так
устроены и Happ, и v2rayNG.

## Сборка APK прямо с телефона (без компьютера)

В репозиторий уже добавлен файл `.github/workflows/build-apk.yml` —
GitHub Actions сам соберёт APK в облаке при каждом пуше. Тебе нужно
только загрузить файлы в GitHub, компьютер не требуется:

1. Установи приложение **GitHub** (App Store / Google Play) или открой
   github.com в браузере телефона — создай новый **публичный или
   приватный репозиторий**, например `gradelvpn-app`.
2. Самый простой способ загрузить сразу все файлы с телефона —
   зайти на страницу репозитория → **Add file → Upload files** и
   перетащить/выбрать файлы по одной папке за раз (`lib/`, `.github/workflows/`,
   `pubspec.yaml`, `README.md`) — веб-интерфейс GitHub открывается и в
   мобильном браузере.
   - Если браузер телефона не даёт выбрать сразу папку — открой каждую
     подпапку (`lib`, `lib/screens`, `lib/services`, `lib/theme`,
     `lib/widgets`, `.github/workflows`) отдельно через **Create new file**
     и вставь путь вида `lib/screens/home_screen.dart` в поле имени — GitHub
     сам создаст нужные папки, а содержимое файла просто вставь текстом.
   - Более удобный вариант — приложение **Termux** (Android) или
     **Working Copy** (iOS): полноценный git-клиент, куда можно
     распаковать присланный архив и одной командой запушить всё разом
     (`git add . && git commit -m "init" && git push`).
3. После пуша в ветку `main` открой вкладку **Actions** в репозитории —
   там появится запуск сборки. Через несколько минут в конце — файл
   **GradelVPN-apk** в разделе Artifacts. Скачиваешь его прямо на
   телефон и ставишь (нужно разрешить установку из неизвестных
   источников в настройках Android).

Workflow сам:
- достраивает недостающую Android-часть проекта (`flutter create .`),
- прописывает нужные разрешения в `AndroidManifest.xml`,
- выставляет `minSdkVersion 21`,
- собирает релизный `.apk`.

Тебе не нужно ничего собирать локально — только один раз положить код
в репозиторий.

## Формат подписки

Приложение ожидает, что бот отдаёт по ссылке (например
`https://c11.play2go.cloud:20080/sub/<токен>`) **base64 от списка
share-ссылок**, разделённых переводом строки — стандарт панелей вроде
3x-ui/Marzban:

```
vless://uuid@server1.com:443?type=tcp&security=tls#Server-1
vless://uuid@server2.com:443?type=tcp&security=tls#Server-2
```

Если бот отдаёт список открытым текстом (не base64) — тоже сработает,
сервис сам определяет формат.

## Структура

- `lib/services/subscription_service.dart` — загрузка и декодирование подписки
- `lib/services/server_store.dart` — хранение подписки и списка серверов между запусками
- `lib/services/v2ray_service.dart` — подключение/отключение, пинг сервера
- `lib/screens/` — главный экран, список серверов, добавление подписки
- `lib/theme/app_theme.dart` — тёмная тема

## Что можно добавить дальше

- Иконка приложения (`flutter_launcher_icons`) — можно тоже прописать в
  workflow, если пришлёшь картинку логотипа
- Экран статистики трафика — данные уже приходят в `V2RayStatus`, просто
  не выведены на экран
- Публикация в Google Play потребует отдельного ревью (VPN-приложения
  проверяют строже)
